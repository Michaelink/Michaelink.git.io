Docker安装和基本使用


uname -r 查看当前内核版本 Docker 要求 Ubuntu 系统的内核版本高于 3.10

卸载旧版本

apt remove docker docker-engine docker.io containerd runc

Ubuntu系统的Aliyun源安装 ( 国内 ):

安装必要的一些系统工具

apt-get -y install apt-transport-https ca-certificates curl software-properties-common

安装 GPG 证书

curl -fsSL http://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | sudo apt-key add -


写入软件源信息

sudo add-apt-repository "deb [arch=amd64] http://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable"
#### 更新并安装 Docker CE apt-get update && apt-get install -y docker-ce

配置阿里云镜像加速

cd /etc/docker

tee /etc/docker/daemon.json <<-‘EOF’
{
“registry-mirrors”: [“https://tldg6byw.mirror.aliyuncs.com"]
}
EOF

systemctl daemon-reload
systemctl restart docker

Install using the script(不建议生产环境使用)

curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

CentOS系统下Docker的Rpm安装

下载Rpm包

地址: https://download.docker.com/linux/centos/7/x86_64/stable/Packages/
docker-ce-17.03.0.ce-1.el7.centos.x86_64.rpm
docker-ce-selinux-17.03.0.ce-1.el7.centos.noarch.rpm

安装

yum update
在rpm包所在目录下执行:
yum -y install *
systemctl start docker
systemctl enable docker
systemctl status docker

拉取centos镜像

docker pull centos

启动容器

docker run -d -it -p 80:80 -p 443:443 -p 8888:8888 centos
-p是端口映射，前面的是本机端口，后面的是容器端口
启动后会返回一个字符串，这个是容器存储的路径

docker ps 查看当前正在运行的docker镜像及id

docker exec -it 容器id /bin/bash 进入容器(退出终端后容器不会退出)
docker attach (退出bash后容器也会停止)
exit 退出容器的终端

docker stop 容器id 停止当前运行的容器

服务器重启,那么容器就会退出

docker ps -a 查看所有的镜像
docker restart 容器id 恢复退出的容器
sudo docker update –restart=always 容器id 设置永久性自启动

systemctl stop docker

service docker start

docker version

docker info

docker关于tomcat的操作

docker pull tomcat
docker rmi tomcat / docker image rm tomcat 删除镜像

docker images

docker system df 查询镜像 、容器、数据卷占用空间

虚悬镜像

中间层镜像

docker run -p 8080:8080 –name tomcat -d tomcat
docker ps
ip a

docker stop 08047bc6008d
docker ps -a
docker start 08047bc6008d
docker rm -f 08047bc6008d
docker exec -it 08047bc6008d /bin/bash

cd /usr/local/tomcat/webapps/ROOT
echo hello world > index.jsp

复制war包到tomcat容器中

docker cp xxx.war tomcat:/usr/local/tomcat/webapps
docker restart 08047bc6008d

tail -f catalina.out

以挂载方式启动

上述执行有个弊端就是容器重启后项目就不在了，我们可以以挂载的方式启动
docker run -d -v xxx.war:/usr/local/tomcat/webapps/xxx.war –name tomcat2 -p 8081:8080 tomcat

chmod +x *.sh

ctrl + d / exit 退出终端

docker container prune 删除所有已停止的容器

Dockerfile 定制镜像

mkdir /usr/local/docker/myshop
vim Dockerfile :
FROM tomcat:latest
COPY index.jsp /usr/local/tomcat/webapps/ROOT

vim index.jsp :
hello world

docker build -t myshop .
docker run -p 8081:8080 –name myshop -d myshop

docker container ls -a
